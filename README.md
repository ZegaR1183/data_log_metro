# Log Analyzer

Инструмент для анализа логов сетевых устройств и генерации отчетов по состоянию оборудования.

## Описание

Этот скрипт:
- Очищает первоначальные данные
- Создает DataFrame для анализа сетевых устройств (Juniper MX, ACX)
- Генерирует детализированные отчеты по:
  - Общему состоянию устройств
  - Состоянию вентиляторов
  - Температуре компонентов
 
Также:
- Сохраняет результаты в Excel-файл
- Подключается к БД PostgreSQL и сохраняет данные
- Создает файл app.log с журналом событий

## Зависимости

- pandas >= 1.3.0
- openpyxl >= 3.0.0
- python >= 3.8
- psycopg2 >= 2.9.0
- sqlalchemy >= 1.4.0
- python-dotenv >= 0.19.0

## Использование
1. Подготовьте лог-файл с именем `result_output` в директории `input_file`
2. Запустите скрипт: `python log_metro.py`
3. Отчет будет сгенерирован в формате Excel с именем `alarm_report_metro_YYYY-MM-DD.xlsx`


### Выходные данные
- `./temp/clear_log.txt` - очищенный лог-файл
- `./output_files/alarm_report_metro_YYYY-MM-DD.xlsx` - сформированный отчёт
- `./logs/app.log` - журнал работы приложения

### Содержание отчета

Отчет в Excel содержит 4 листа:
- **Общая информация** - таблица со всеми устройствами и значениями параметров
- **Статистика** - показывает проблемы и количество устройств
- **Вентиляторы** - таблица устройств с отключенными вентиляторами
- **Температура** - таблица устройств с высокой температурой (по умолчанию > 50°C)
